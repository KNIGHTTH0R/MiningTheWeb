Bars in NYC - Doh Y Jung

This was an example I found of bars distributed in NYC based on the Liquor license information. I found this to be a pretty poorly constructed map. First of all, it uses Flash. Secondly, crowded data points does not aggregate to help viewers get a better sense of the distribution. There is no legend explaining what each colors and symbols mean.

So I tried to create a better version using cartodb.

Instead of showing all the locations with liquor store license, I've filtered down only what people would consider as "Bars" by filtering the license type. And I also excluded listings with no business names.

Visualizing @BrooklynBrewery Engagement - Doh Y Jung

In keeping with the theme of bars, I first wanted to try using CartoDB with a dataset that I am familiar with. So from Brandwatch I exported all the "Engagement" (Retweet or At mention) data to @BrooklynBrewery. This CartoDB map is showing geographically where people POST on Twitter reaching out to @BrooklynBrewery.

The static set of data was taken from the past 7 days (Sep 7th 2015 - Sep 13th 2015). I tried to provide certain level of interactivity by showing the actual tweet upon clicking dot representation. There was a intensity display through the green gradient.

If you zoom into North America, you can see that more people tweet to @BrooklynBrewery from the East Coast, and globally the brand seems to have some traction from the U.K.

Map Hacking the Sisyphyus Subway map - Richard Dunks

This is an example visualization critique of a great visualization created by the master of interactive visualization and design, Richard Dunks. In this latest incarnation, he has remade a visualization from the Citizens Budget Commission using CartoDB, Leaflet, D3.js, and jQuery. While the original visualization showed the data in an interactive format using Tableau Public, Richard created a custom JavaScript map and was able to better display the data, providing multiple data layers and better interactivity with the data.
